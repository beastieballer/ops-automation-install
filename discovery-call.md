# Ops Automation Install — Discovery Call Script

## Goal of the Call
Figure out four things:
1. where browser-based admin work is creating drag
2. whether there is one strong workflow worth automating first
3. what guardrails are required
4. whether the buyer is ready for a practical install now

This should feel like an operating conversation, not an AI lecture.

---

## Ideal Buyer
- contractor
- service business owner
- founder-led team
- operations lead
- admin-heavy small team using Gmail, dashboards, portals, spreadsheets, CRMs, or industry SaaS tools

---

## Target Call Length
- 20–30 minutes for qualification
- 30–45 minutes if mapping the workflow live

---

## Opening Frame
Use something like:

> I’m not here to sell you generic AI. I want to understand where browser-based admin work is eating time in your business, where things get missed or delayed, and whether there’s one workflow we can install cleanly to reduce that drag.

---

## Section 1 — Surface the pain
Ask:
- What admin or coordination work takes more time than it should right now?
- Where are you or your team bouncing between tabs just to keep things moving?
- What repetitive work shows up every day or every week?
- Where do things most often get missed, delayed, or forgotten?
- What tasks feel too small to hire for but too expensive to keep doing manually?

Listen for:
- inbox overload
- manual follow-up
- recurring reporting
- status chasing
- copying details from one system to another
- vendor/admin friction

---

## Section 2 — Map the workflow
Ask:
- What kicks off the workflow?
- What systems are involved?
- What information are you pulling from each one?
- What decisions are being made during the process?
- What output do you need at the end?
- Where does a human absolutely need to review before anything is sent or changed?

Capture:
- trigger
- inputs
- decisions
- outputs
- exceptions
- risk points

---

## Section 3 — Validate fit for automation
Ask:
- Is this workflow repetitive enough that the steps are usually similar?
- Is it mostly browser-based today?
- How often does it happen?
- If we improved this in the next week, what would that change for you?
- What would count as a clear win?

Good first workflows are:
- repetitive
- browser-heavy
- easy to verify
- painful enough to matter
- narrow enough to scope quickly

---

## Section 4 — Guardrails and risk
Ask:
- What actions should never happen without human review?
- Are there any systems or pages you would not want included?
- Any customer, financial, or compliance sensitivities?
- What kind of mistakes would be unacceptable?

Reinforce:
- we automate triage, extraction, summaries, and draft prep first
- we keep high-trust actions reviewed by a human
- we do not treat this like reckless full autonomy

---

## Section 5 — Value and urgency
Ask:
- How much time do you think this workflow costs each week?
- Is the bigger pain time loss, missed details, slow response, or admin overload?
- Why does solving this matter now?
- If this worked well, where would you want to expand next?

---

## Section 6 — Close toward the install
If fit is strong:

> I think there’s a clean first install here. The right next step would be an Ops Automation Install where I scope the workflow, set the guardrails, build it, test it, and get it running in a way your team can actually use.

If fit is medium:

> I think there’s something here, but I’d want to narrow the workflow a little more so the first install is tight and high-confidence.

If fit is weak:

> I don’t think the first move is automation yet. I think you’d benefit from clarifying the process ownership and workflow steps first so we’re not automating chaos.

---

## Qualification Checklist
Strong signals:
- can describe a repetitive workflow clearly
- real time is being lost today
- browser is the main operating surface
- decision-maker is present
- comfortable with a scoped paid install
- understands the need for review guardrails

Weak signals:
- wants “AI ideas” but no concrete pain
- wants a custom platform immediately
- no access to systems or process owners
- expects zero-review automation for sensitive work
- workflow is too vague or too chaotic to scope

---

## Good First Install Candidates
- inbox triage + action summary
- dashboard/report capture + summary
- vendor or billing follow-up prep
- job/admin status update workflow
- notes-to-actions workflow
- recurring operational summary workflow

## Weak First Install Candidates
- fully autonomous client communication
- mission-critical financial actions
- giant cross-team orchestration from day one
- workflows with no stable steps or no owner
